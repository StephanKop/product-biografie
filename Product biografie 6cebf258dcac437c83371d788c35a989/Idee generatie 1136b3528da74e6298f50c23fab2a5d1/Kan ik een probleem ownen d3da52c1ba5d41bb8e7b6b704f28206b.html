Kan ik een probleem ownen?

Tijdens mijn groenlicht presentatie kwam david met de vraag. Kan je een probleem ownen? Deze vraag kwam voort uit een bepaald user scenario.

Namelijk als een developer oordeelt dat hij een probleem aan bijvoorbeeld een server zelfstandig en snel op kan lossen. Dan kan het zijn dat hij dit niet deelt met zijn collega's en daardoor misschien een andere developer exact hetzelfde idee heeft om hetzelfde probleem te fixen.

In dit geval zou het handig kunnen zijn dat developers op het dashboard aan kunnen geven dat zei een probleem oppakken. Ik vond dit een interessant gegeven en beloot dit dus even kort te onderzoeken. Ik stelde mezelf namelijk de vraag, valt dit binnen mijn scope of erbuiten. Wordt het dashboard door deze functie nou meer een uit de kluiten gewassen ticket systeem of past het binnen de scope van een dashboard.

Ik kan kort teruggrijpen naar mijn onderzoeksvraag: Wat voor dashboard heb ik nodig?

Die vraag heb ik beantwoord door te kiezen voor een operationeel dashboard. Dit houd in dat het voornamelijk gefocust is op actuele gebeurtenissen en operationele processen. Een operationeel dashboard wordt voornamelijk gebruikt om real time data te tonen. Een ticket systeem past hier dus eigenlijk niet echt bij. Het tonen van de tickets kan prima omdat dat onder de realtime data valt. Echter wanneer je een combinatie gaat maken tussen een operationeel dashboard en een tactical dashboard is het mogelijk om dit soort "ticket" zaken ook te regelen. Heel conventioneel is dit echter niet en tijdens deskresearch had ik moeite met het vinden van combinaties tussen deze twee dashboard vormen.
