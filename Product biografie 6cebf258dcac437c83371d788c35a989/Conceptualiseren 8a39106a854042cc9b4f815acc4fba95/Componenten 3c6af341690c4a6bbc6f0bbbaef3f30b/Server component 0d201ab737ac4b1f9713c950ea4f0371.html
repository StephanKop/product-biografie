Waar ga ik de data vandaan halen?

Dit is een vraagstuk dat ik op moet pakken met psxtreme en bob van leeuwen. De data van de server moet opgehaald worden vanaf de vps servers zelf dus het lastige hieraan is dat er verschillende partijen aan verbonden zijn.

Hoe toon ik de informatie?

De informatie zal getoond worden in de front-end door middel van .map() functie. De .map() functie loopt over het servers array heen. Voor elk elk element maakt hij dan de onderstaande html code aan en vult die met de data uit het array.

Om de status van de servers te veranderen maak ik gebruik van een object.keys functie met een relatief grote if statement erin. 

function serverStatus() {
        Object.keys(servers).map((i) => {
            if (servers[i].status === "Unavailable") {

                let statusImage = document.querySelectorAll('#statusIcon' + servers[i].id)!;
                statusImage.forEach(function (el) {
                    el.setAttribute('src', require("../img/error-icon.svg"));
                });

                let statusMessage = document.querySelectorAll('#status' + servers[i].id)!;
                statusMessage.forEach(function (el) {
                    if (el.innerHTML === "Unavailable") {
                        el.classList.add('error');
                    }
                });

                let statusMemory = document.querySelectorAll('#memory' + servers[i].id)!;
                statusMemory.forEach(function (el) {
                    if (el.innerHTML === "Error") {
                        el.classList.add('error');
                    }
                });

                let statusDisk = document.querySelectorAll('#disk' + servers[i].id)!;
                statusDisk.forEach(function (el) {
                    if (el.innerHTML === "Error") {
                        el.classList.add('error');
                    }
                });
            } else {
                let statusImage = document.querySelectorAll('#statusIcon' + servers[i].id)!;
                statusImage.forEach(function (el) {
                    el.setAttribute('src', require("../img/statusok.png"));
                });

                let statusMessage = document.querySelectorAll('#status' + servers[i].id)!;
                statusMessage.forEach(function (el) {
                    el.classList.remove('error');
                });

                let statusMemory = document.querySelectorAll('#memory' + servers[i].id)!;
                statusMemory.forEach(function (el) {
                    el.classList.remove('error');
                });

                let statusDisk = document.querySelectorAll('#disk' + servers[i].id)!;
                statusDisk.forEach(function (el) {
                    el.classList.remove('error');
                });
            }
        });
    }

Hoe ga ik de data doorgeven aan de sub pagina's?

https://reactjs.org/docs/components-and-props.html

De detailpagina gaat opgebouwd worden door data door te geven aan een onderliggend component met props van react. Hierdoor kan dan de data in het onderliggende component getoont worden na het klikken op een server. Deze props maak ik dan dynamisch waardoor elk server element linkt naar de juiste data.

voorlopige technische flow

De data gaat opgehaald worden vanaf de server, de backend exposed deze data dan in de restapi waaruit ik in de front-end de components kan opbouwen en kan tonen op de gewenste situaties.

De informatie zal dan worden doorgegeven naar het detail pagina component via props in react.

Iteratie1

  • De voornaamste feedback kwam neer op de onduidelijkheid over de aflopende balkjes. Deze heb ik dus veranderd naar balken die zich altijd op 100% van de lengte tonen maar gevuld worden als een progress bar. Zo is het voor de developers duidelijker waar de deadline naar toe loopt omdat de 0% en 100% waarde van de balk duidelijk zijn.
  • Ook heb ik de deadline datum veranderd naar dagen resterend zodat de developers niet meer voor hunzelf hoeven omrekenen hoeveel dagen er dan resterend zijn.
  • Naar wens van de developers heb ik ook een sorteerfunctie toegevoegd waarmee deadlines op naam en datum gesorteerd kunnen worden.
