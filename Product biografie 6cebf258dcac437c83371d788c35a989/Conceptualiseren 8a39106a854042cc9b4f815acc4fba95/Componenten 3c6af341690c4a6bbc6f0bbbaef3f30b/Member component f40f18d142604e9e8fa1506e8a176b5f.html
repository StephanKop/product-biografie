Hoe koppel ik members aan deadlines

In de deadlines enitity wordt de member entity aangeroepen zodat de member data in de deadline data komt te staan. Door gebruik te maken van een many to many relatie kan ik via deadlines members ophalen, en via members deadlines aanroepen.

@ManyToMany(type => MemberEntity)
    @JoinTable()
    members: MemberEntity[];

Hoe stel ik alert waardes in wanneer een member afwezig is?

Doordat ik aan het members object een "present" waarde toevoeg met een boolean die in de front-end veranderd kan worden. Kan ik checken met een if statement of de de user aanwezig is of niet. De functie hieronder stelt een alert kleur in en zet een class op een member afbeelding.

const members = deadlines[i].members;
        members.forEach(function (el) {
                if (el.present === false) {
                    const unavailableMember = document.querySelectorAll("#deadline" + deadlines[i].id);
                    const unavailableMemberImage = document.querySelectorAll("#presentfalse");
                    unavailableMemberImage.forEach(function (el) {
                        el.classList.add('unavailableMember');
                    })
                    // @ts-ignore
                    unavailableMember.forEach(function (el:HTMLElement) {
                        el.style.background = "linear-gradient(90deg, rgba(255,181,0,1) 0%, rgba(255,181,0,1)" + (100 - deadlineLength(deadlines[i])) + '%' + ", rgba(255,255,255,1)" + (100 - deadlineLength(deadlines[i])) + '%' + ")";
                    })
                }
            })
        })

Design

Iteratie 1

Het member component is een relatief simpel component in card style die alle members weergeeft met een afbeelding, hun naam en de aanwezigheids status. Via een hover animatie komt de edit knop tevoorschijn en kan de status via een popup modal aangepast worden.

Iteratie 2

De status van een afwezig personeelslid is gewijzigd naar een transparente state. Uit de feedback sessie blijkt dat de devs het belangrijker vinden om te zien wie de aanwezige personeelsleden zijn dan te zien wie er afwezig is. Bij andere functies als de deadlines is het fijn dat een afwezig personeelslid een alert functie triggered. Echter in het member component is dat overbodig en staan de aanwezige personeelsleden centraal. Door dus de opacity van de afwezige personeelsleden aan te passen veranderd de visuele hierarchie en verschuift de focus naar de aanwezige personeelsleden.
