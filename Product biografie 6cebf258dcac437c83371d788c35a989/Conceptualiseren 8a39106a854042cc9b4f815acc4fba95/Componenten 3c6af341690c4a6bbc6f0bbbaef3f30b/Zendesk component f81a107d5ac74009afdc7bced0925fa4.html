Zendesk component

Doel van het component

  • De developers inlichten over de status van openstaande tickets
  • De developers inlichten over nieuw binnengekomen tickets

Techniek

Dit component maakt gebruik van de zendesk api.

Technische obstakels

Hoe werkt de zendesk api?

Hoe laat ik een notificatie afgaan als er een nieuw ticket komt?

Hoe combineer ik de user api call met de tickets api call op de backend?

Technische flow

backend maakt een call op de zendesk api met een aantal parameters. 

const url = 'https://gillztest.zendesk.com/api/v2/tickets.json?sort_by=id&sort_order=desc'

De front-end maakt dan een call op de backend en renderd de data uit met een .map() functie over het array.

Tickets api geeft alleen user id's terug

De tickets api geeft in de response body alleen user id's terug. Op de front-end heb ik uiteraard weinig aan id's aangezien ik in de interface namen moet tonen. Als oplossing hierop heb ik bedacht om 2 calls naar de zendesk api uit te voeren.

  1. Api call voor tickets
  1. Api call voor users

Hierna voeg ik door de id's en de user data te linken alles samen in 1 array die ik dan return op mn rest api.

@Get()
  async getZendesk() {
    const tickets = await this.zendeskService.getZendesk();
    const users = await this.zendeskService.getZendeskUser();
    return tickets["tickets"].map(ticket => ({
      ...users["users"].find((user) => (user.id === ticket.assignee_id)),
      ...ticket
    }));
  }

Priority kleur functie

Aan de hand van de priority waarde die in de tickets state staat van de front-end kan ik met een if statement verschillende classes toewijzen door middel van dom manipulation. 

function priorityColor() {
        Object.keys(tickets).map((i) => {
            if (tickets[i].priority === 'high') {
                let highTickets = document.querySelectorAll("#priorityid" + tickets[i].id);
                highTickets.forEach(function (el) {
                    el.removeAttribute("class");
                    el.classList.add('high');
                });
            } else if (tickets[i].priority === 'urgent') {
                let highTickets = document.querySelectorAll("#priorityid" + tickets[i].id);
                highTickets.forEach(function (el) {
                    el.removeAttribute("class");
                    el.classList.add('urgent');
                });
            } else if (tickets[i].priority === 'normal') {
                let highTickets = document.querySelectorAll("#priorityid" + tickets[i].id);
                highTickets.forEach(function (el) {
                    el.removeAttribute("class");
                    el.classList.add('normal');
                });
            } else if (tickets[i].priority === 'low') {
                let highTickets = document.querySelectorAll("#priorityid" + tickets[i].id);
                highTickets.forEach(function (el) {
                    el.removeAttribute("class");
                    el.classList.add('low');
                });
            }
        })
    }

Design

Het eerste ontwerp is net zoals het build component een list view. Dit is de meest overzichtelijke manier om een grote hoeveelheid data te tonen op een dashboard. De status badge veranderd van kleur wanneer de status wijzigt. Waiting for reply is groen want dan wordt er van de Dev geen actie meer vereist. Open is rood omdat er een actie wordt vereist van de Dev en de solved status is grijs.
