Eerst is het belangrijk om te kiezen tussen een relationele of een non rationele database. Om deze keuze te maken doe ik onderzoek naar de verschillen en de implementatie die ik nodig ga hebben tijdens dit project.

Relationele/sql database

Een Relationele database wordt opgebouwd uit verschillende tabellen met rijen. Je kunt echter binnen een relationele database relaties leggen tussen verschillende tabellen. zie het voorbeeld hieronder.

Binnen mijn project is het bijvoorbeeld handig om werknemers te koppelen aan projecten/deadlines. Om dit te verwezenlijken kan dus het beste een koppel tabel opgezet worden waarin de project id en de member id aan elkaar gekoppeld worden. Hierdoor kan het via 1 query opgevraagd worden uit de database en krijg je altijd een goede dataset terug. Een nadeel om te overwegen is dat een relationele database wel een stuk extra complexiteit toevoegt.

Voorbeelden van relationele databases die gebruikt worden in combinatie met NestJS.

  • SQL Server Express
  • PostgreSQL
  • SQLite
  • MySQL
  • MariaDB

opsomming citaat van relationele databases:

  • They work with structured data.
  • Relationships in the system have constraints, which promotes a high level of data integrity.
  • There are limitless indexing capabilities, which results in faster query response times.
  • They are excellent at keeping data transactions secure.
  • They provide the ability to write complex SQL queries for data analysis and reporting.
  • Their models can ensure and enforce business rules at the data layer adding a level of data integrity not found in a non-relational database.
  • They are table and row oriented.
  • They Use SQL (structured query language) for shaping and manipulating data, which is very powerful.
  • SQL database examples: MySql, Oracle, Sqlite, Postgres and MS-SQL. NoSQL database examples: MongoDB, BigTable, Redis, RavenDb, Cassandra, Hbase, Neo4j and CouchDb.
  • SQL databases are best fit for heavy duty transactional type applications.

non relationele / nosql database

nosql databases maken in tegenstelling tot relationele databases geen gebruik van tabellen met data maar van documenten. Deze documenten kunnen veel verschillende vormen van data opslaan en worden dus vaak gebruikt wanneer er een grote hoeveelheid data dat geen betrekking tot elkaar heeft opgeslagen moet worden.

Key-value stores:

De simpelste vorm van een nosql database. Hierin wordt data opgeslagen via key en value paren. Door de key op te zoeken in de database krijg je de value terug. Simpelere vormen van databases bestaan niet.

Document stores:

deze zijn iets complexer dan de key-value stores maar zijn ongeveer hetzelfde. In deze vorm kan echter een key verschillende vormen aannemen. Een key kan bijvoorbeeld een array met objecten bevatten.

Graph databases:

Dit is de meest geavanceerde vorm en nieuwste vorm van nosql databases. Dit is een soort hybride vorm tussen een nosql en sql database. Er kunnen namelijk relaties worden gelegd tussen verschillende onderdelen.

opsomming nosql database functionaliteiten:

  • They have the ability to store large amounts of data with little structure.
  • They provide scalability and flexibility to meet changing business requirements.
  • They provide schema-free or schema-on-read options.
  • They have the ability to capture all types of data “Big Data” including unstructured data.
  • They are document oriented.
  • NoSQL or non-relational databases examples:MongoDB, Apache Cassandra, Redis, Couchbase and Apache HBase.
  • They are best for Rapid Application Development. NoSQL is the best selection for flexible data storage with little to no structure limitations.
  • They provide flexible data model with the ability to easily store and combine data of any structure without the need to modify a schema.

ORM

https://blog.bitsrc.io/what-is-an-orm-and-why-you-should-use-it-b2b6f75f5e2a

ORM betekend object relational mapper. Kort samengevat is een orm een laag die ervoor zorgt dat je in je eigen codetaal database query's kan uitvoeren. Zonder orm zal je dus normale sql queries uit moeten voeren.

Voorbeeld sql query: 

SELECT * FROM users WHERE email = 'test@test.com';

Voorbeeld orm query in javascript: 

var orm = require('generic-orm-libarry');
var user = orm("users").where({ email: 'test@test.com' });

Er zijn verschillende ORM packages op npm om te gebruiken maar TypeORM is de grootst ondersteunde en past het beste bij het typescript van NestJS.

Typeorm

Voor de backend maak ik gebruik van typeOrm. lees hier de research naar een orm. Voor mijn data heb ik verschillende crud acties nodig. Zo maak ik in mijn applicatie gebruik van: Create, Update en Delete crud acties.

Neem het voorbeeld van de deadlines:

  • Een deadline moet in de applicatie aangemaakt kunnen worden. (Create)
  • Een deadline moet bewerkt kunnen worden. Zo moet namelijk de naam, deadline datum, en de members veranderd kunnen worden. (Update)
  • Een deadline moet kunnen worden verwijderd wanneer deze is komen te vervallen. (Delete)

Typeorm maakt het mij makkelijk om in de backend gemakkelijk de gewenste acties voor de juiste routes aan te maken.
